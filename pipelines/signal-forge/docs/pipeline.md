# Pipeline

## Objective

Signal Forge의 목적은 투자 아이디어를 즉흥적으로 따라가는 대신,
반복 가능한 데이터 파이프라인으로 구조 변화의 신호를 먼저 포착하는 것입니다.

핵심 질문은 아래 다섯 가지입니다.

- 지금 한국 증시에 유입되는 자금은 늘고 있는가
- 반도체 업황은 기대가 아니라 숫자로 개선되고 있는가
- 정책 변화는 발표 단계인가, 실제 시행 단계인가
- 외국인 자금 유입 논리는 현실적인 실행 조건을 갖췄는가
- 현재 강세가 단기 뉴스인지 구조 변화인지 구분할 수 있는가

## System Shape

### 1. Collect

매일 또는 매주 원천 데이터를 수집합니다.

- 시장 데이터: 지수, 수급, 거래대금, 환율, 금리
- 정책 데이터: 금융위, 거래소, 한국은행, 기재부, MSCI 관련 문서
- 산업 데이터: 반도체 가격, AI 인프라 투자, 조선/방산/전력기기/원전 뉴스
- 기업 데이터: 실적, 컨센서스 변화, 자사주 소각, CAPEX, 수주

산출물:

- 원천 응답 저장
- 표준 스키마로 정규화된 행 데이터
- 문서형 데이터의 요약 및 태그

### 2. Normalize

서로 다른 소스의 형식을 공통 스키마로 맞춥니다.

- 날짜, 종목 코드, 기관명 표준화
- 중복 기사 및 중복 공시 제거
- 단위 통일
- 신뢰도 메타데이터 추가

산출물:

- Supabase 적재용 정규화 레코드
- 추적 가능한 `source_url`, `published_at`, `collected_at`

### 3. Score

수집된 데이터를 테마별 신호로 바꿉니다.

- 반도체 엔진
- 정책/제도 엔진
- 수급 엔진
- 유동성/머니무브 엔진
- AI 인프라 엔진

각 엔진은 점수와 근거를 함께 남깁니다.

- `score`
- `delta`
- `evidence`
- `confidence`

### 4. Brief

주간 또는 일간 브리프를 생성합니다.

- 이번 주 핵심 변화
- 강해진 주장과 약해진 주장
- 체크해야 할 리스크
- 영상 소재 후보 3개

산출물:

- Markdown 리포트
- 대본 초안 입력용 구조화 텍스트

## Execution Plan

### GitHub

- 수집기 코드와 SQL 마이그레이션 관리
- 문서와 템플릿 버전 관리
- 이슈와 TODO 추적

### GitHub Actions

- 스케줄 기반 수집 실행
- 적재 실패 감지
- 주간 브리프 생성
- 선택적으로 요약 결과를 아티팩트로 업로드

### Supabase

- 정규화 테이블 저장소
- 문서형 데이터 저장
- 검색용 메타데이터 및 임베딩 저장
- 대시보드/API 백엔드

## Recommended Cadence

- 매일 장 마감 후: 시장/수급/뉴스 수집
- 매주 주말: 정책/산업/기업 브리프 재계산
- 매주 1회: 영상 후보 주제 자동 생성

## Minimal First Version

처음부터 모든 걸 자동화하지 않습니다.

1. 핵심 종목 10개 추적
2. 거시/수급 지표 10개 추적
3. 정책 문서 5개 소스 추적
4. 주간 Markdown 브리프 생성
5. 이후 점수화와 대본 자동화 확장
