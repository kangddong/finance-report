# Signal Forge Guide

이 파일은 백과사전이 아니라 목차입니다. 실제 운영 지침은 `docs/` 아래 문서를 기준으로 봅니다.

## Start Here

- `docs/pipeline.md`: 수집, 정제, 점수화, 브리프 생성까지의 전체 파이프라인
- `docs/data-inventory.md`: 어떤 데이터를 왜 모으는지에 대한 수집 목록

## Working Rules

- 루트에는 진입점 문서와 실행 스크립트만 둡니다.
- 상세 정책, 운영 가이드, 데이터 정의는 `docs/` 아래로 내립니다.
- 자동 생성 산출물과 임시 수집 파일은 Git에 넣지 않습니다.
- 새로운 자동화는 "무엇을 수집할지"보다 "어떤 질문에 답할지"를 먼저 문서화합니다.

## Near-Term Goal

이 저장소의 1차 목표는 `GitHub + GitHub Actions + Supabase` 조합으로
시장/정책/산업/기업 데이터를 누적하고, 주간 투자 브리프 초안을 자동으로 만드는 것입니다.
