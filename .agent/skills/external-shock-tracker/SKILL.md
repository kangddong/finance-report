# External Shock Tracker

## 개요

외부 충격 이벤트를 단순 뉴스 목록이 아니라 시장 영향 관점으로 정리한다.
이벤트 자체, 충격 경로, 자산 가격 반응, 후속 체크포인트를 함께 적는다.

## 기본 흐름

1. 기간과 범위를 먼저 확정한다.
2. 이벤트를 `유가 / 환율 / 전쟁 / 정책 / 규제 / 정치 이벤트`로 분류한다.
3. 날짜, 제목, 핵심 내용, 시장 영향, 후속 관전 포인트를 정리한다.
4. 같은 사건의 중복 기사를 제거하고 최초 충격과 후속 대응을 분리한다.
5. 최종 결과는 날짜순 타임라인으로 정리한다.

## 수집 원칙

- 반드시 실제 날짜를 적는다.
- `사건 발생일`과 `기사 작성일`이 다르면 사건 발생일을 우선한다.
- 유가와 환율은 숫자 변화보다 왜 움직였는지의 원인을 적는다.
- 정책과 규제는 `발표`, `시행`, `법원 판단`, `당국 발언`을 구분한다.
- 전쟁과 지정학 이벤트는 공급망, 해상 운송, 안전자산 선호로 이어지는 경로를 같이 쓴다.
- 정치 이벤트는 선거, 관세, 외교 갈등, 정상 발언처럼 시장 민감도가 큰 것만 고른다.
- 최신 또는 기간형 요청은 웹에서 다시 확인한다.

## 기본 출력 형식

```markdown
# 외부 충격 이벤트 타임라인

## [YYYY-MM-DD] [카테고리]
- 이벤트:
- 핵심 내용:
- 시장 영향:
- 후속 체크포인트:
- 출처:
```

## 카테고리 정의

- `유가`: 원유 가격 급등락, 공급 차질, 비축유 방출, OPEC 관련 충격
- `환율`: 달러 강세/약세, 원화 급락, 안전자산 선호, 중앙은행 영향
- `전쟁`: 군사 충돌, 공격, 제재, 해상 운송 차질
- `정책`: 금리, 관세, 재정정책, 정부 발표
- `규제`: 법원 판단, 감독당국 조치, 산업 규제 변화
- `정치 이벤트`: 선거, 정상 발언, 외교 갈등, 정권 교체 리스크

## 품질 기준

- 뉴스 헤드라인만 옮기지 말고 시장 충격 경로를 설명한다.
- 같은 이슈는 최초 충격과 정책 대응을 분리해서 기록한다.
- 근거가 약한 이벤트는 제외하거나 불확실성을 표시한다.
- 기간 전체를 다룰 때는 중요도가 높은 이벤트부터 빠짐없이 정리한다.
